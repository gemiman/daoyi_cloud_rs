# Rust 重构任务清单（Salvo 0.85.0 + redis 1.0.0-rc.4 + nacos-sdk 0.5.3 + sea-orm 2.0.0-rc.19）

## 目标与原则
- [ ] Rust 版接口契约 1:1 兼容（路径/方法/入参/出参/鉴权/租户/缓存/MQ/副作用），可平滑替换 Java。
- [ ] 逐模块迁移、保持可运行基线、可回滚；敏感配置放 `config.local.toml` 或环境变量。
- [x] 交付形态：Rust workspace，lib crate 公共复用；每业务模块有独立 `bin`（如 `bin/system-server`、`bin/infra-server` 等）可单独运行，同时提供聚合 `bin/daoyi-server` 汇总全部模块，运行/部署方式与 Java 模块化一致。

## 阶段 0：仓库与目录起手
- [x] 创建 workspace 目录结构并与 Java 模块一一映射：`core/`、`infra/`、`gateway/`、`domain-system/`、`domain-infra/`、`domain-member/`、`domain-mall/`（拆 product/promotion/trade/statistics 子域目录）、`domain-pay/`、`domain-mp/`、`domain-bpm/`、`domain-crm/`、`domain-erp/`、`domain-iot/`、`domain-ai/`、`domain-report/`、`bin/daoyi-server/`，路由前缀/包名与 Java 模块保持同名。
- [ ] 更新根 `Cargo.toml`：声明 workspace 成员与 resolver = "2"，公共 dependency 版本统一。
- [ ] 复制/整理 Java `sql/` 到 `migration/` 作为初始迁移输入；保留 `assets/`、`views/`、`config-example.toml`。
- [ ] 新建 `.env.example` 或 `config.local.example.toml`，列出敏感配置键（db/redis/nacos/jwt/oss/xxl-job/sms/mail/third-party）。
- [ ] 增加 `justfile`/`Makefile`：`just dev|test|lint|migrate|fmt`，README 补充快速开始命令。

## 阶段 1：基础设施空壳跑通
- [ ] 在 `core`：错误码枚举、`ApiResult<T>`、全局错误处理、分页/时间类型封装、JWT 工具、密码哈希、雪花/UUID 工具。
- [ ] 在 `infra`：
  - [ ] Nacos 客户端封装：命名空间/分组/监听，配置反序列化到 `AppConfig`，支持热更新回调。
  - [ ] Redis 客户端：连接池、序列化工具、分布式锁、pubsub 封装。
  - [ ] DB：SeaORM 连接池、多数据源管理，`migration/` 框架落地。
  - [ ] 日志与链路：tracing-subscriber、JSON 日志、可选 otel 导出。
- [ ] 在 `gateway`：Salvo Server/Router 搭建（admin/app 前缀、版本前缀），中间件链：Cors、RequestId、Compression、Timeout、Tracing、Body limit。
- [ ] 在 `bin/daoyi-server`：入口 main，加载配置、初始化 infra（db/redis/nacos/logging）、挂载路由，提供健康检查路由。

## 阶段 2：安全与网关能力
- [ ] 认证：账号密码、社交登录、OAuth2；JWT 多端令牌区分；单点注销；多租户上下文传递（header/param/token）。
- [ ] 授权：RBAC（角色/菜单/数据权限），后台/前台权限拦截；DataScope 查询过滤器。
- [ ] 防护：验证码、短信/邮箱验证、限流（IP/用户/接口）、XSS/CSRF 过滤、上传白名单。
- [ ] 审计：访问/操作/登录日志中间件落库，替代 ApiAccessLog/ApiErrorLog。

## 阶段 3：数据与实体
- [ ] 将 `sql/` 转 SeaORM Migration，保持表/索引/字段一致；生成实体 `entity/` 代码。
- [ ] 建仓储层：公共 CRUD、分页、排序、乐观锁/软删除支持。
- [ ] DTO/VO/BO 映射：validator 规则、serde 序列化，`From/Into`/手写转换替代 MapStruct。

## 阶段 4：公共业务能力
- [ ] 短信/邮件/站内通知（渠道/模板/日志/回调）。
- [ ] 文件/对象存储（本地/NAS/云）上传、预签名、配置管理。
- [ ] Codegen 生成器：输出 SeaORM/Salvo 模板；动态数据源切换与健康检查。
- [ ] 任务/规则：xxl-job 适配或自建调度；IOT 规则、BPM 表达式执行器。
